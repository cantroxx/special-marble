# 특산물 마블 — 퀴즈타운 연동 배포 가이드

낱말대전(`/wordbattle/`)과 **똑같은 방식**으로 붙입니다.
특산물마블을 퀴즈타운 `public/marble/` 에 넣으면 **로그인·랭킹이 자동 공유**돼요.

> ⚠️ 운영(학생 실데이터) 서비스입니다. 아래는 **선생님이 직접** 검토·배포하세요.
> 저(클라우드)는 님 Firebase에 배포/테스트를 할 수 없어, 코드만 만들어 드립니다.

---

## 0) 준비: 특산물마블 정적 빌드
특산물마블 프로젝트(`special-storage`)에서:
```
npm install
npm run build        # dist/ 에 정적 파일 생성 (base:'./' 라 /marble/ 에서도 동작)
```

## 1) 퀴즈타운에 넣기 — `public/marble/`
`special-storage/dist/` 안의 **모든 파일**을 퀴즈타운의
`gas-quiz-firebase/public/marble/` 폴더로 복사합니다.
```
# 예시 (경로는 각자 맞게)
mkdir -p /Users/kdw/Projects/gas-quiz-firebase/public/marble
cp -R /Users/kdw/Projects/special-storage/dist/* /Users/kdw/Projects/gas-quiz-firebase/public/marble/
```
→ 그러면 `https://dj48-quiztown-firebase.web.app/marble/` 로 열려요.
（marble의 index.html이 `/__/firebase/...` 를 로드 → 같은 도메인이라 **로그인·Firestore 자동 공유**）

## 2) 랭킹 광장 탭 — `mb-ranking-plaza.js`
이 폴더의 `public/js/mb-ranking-plaza.js` 를 퀴즈타운 `public/js/` 에 복사하고,
퀴즈타운 `public/index.html` 에서 **wb-ranking-plaza.js 를 로드하는 줄 바로 아래**에 추가:
```html
<script src="js/wb-ranking-plaza.js"></script>
<script src="js/mb-ranking-plaza.js"></script>   <!-- ← 이 줄 추가 -->
```
→ 랭킹 광장 탭 줄에 **"특산물 마블"** 탭이 생기고 `marbleRanking` 순위를 보여줘요.

## 3) 진입 링크 — 학교 메뉴
퀴즈타운 `public/index.html` 의 학교(school) 영역, 낱말대전 링크
(`<a href="/wordbattle/">`) 근처에 아래를 추가:
```html
<a href="/marble/">🧑‍🌾 특산물 마블</a>
```

## 4) 보안 규칙 — `firestore.rules`
`firestore.rules.추가분.txt` 의 두 블록(`marbleRooms`, `marbleRanking`)을
퀴즈타운 `firestore.rules` 안 `match /databases/{database}/documents { ... }` **안쪽에 추가**.

## 5) 검증 + 배포
```
npm run check                      # (퀴즈타운) 정적 검사 통과 확인
firebase deploy --only hosting,firestore:rules
```

## 데이터 구조 (참고)
- `marbleRooms/{4자리코드}` = `{ code, title, status, createdAt, seats:[{uid,name},{uid,name}], battle }`
  - `battle` = 대전 상태 전체(두 플레이어·공유 시장). 액션은 `runTransaction`으로 안전 적용.
- `marbleRanking/{uid}` = `{ name, total, games, wins, updatedAt }` — 승 +30/패 +5 누적(본인만 write)

## 동작 확인 순서
1. 퀴즈타운 로그인 → `/marble/` 접속 → ⚔️ 대전 탭에 **🌐 온라인 대전** 박스가 보이면 로그인·Firebase 연결 성공
2. A 학생 "방 만들기" → 방 번호 확인 → B 학생 "입장(번호)" → A가 "대전 시작"
3. 번갈아 플레이 → 끝나면 승패가 `marbleRanking` 에 기록 → 랭킹 광장 "특산물 마블" 탭에서 순위 확인

## 알아둘 점 (정직하게)
- 온라인 동기화 코드는 낱말대전 net.js 패턴을 그대로 따랐지만, **저는 실제 Firebase에서 테스트를 못 했어요.**
  첫 배포 후 **실제로 두 기기에서 한 판** 돌려보고, 어색한 부분(턴 반영 지연, 방 목록 등)은 알려주시면 바로 고칠게요.
- v1은 클라이언트 트랜잭션 방식(낱말대전과 동일). 부정 방지 강화는 Cloud Functions로 다음 단계.
